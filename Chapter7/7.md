# 7. 에이전틱 시스템의 학습
- 성능 향상하도록 만드는 기능이 에이전트 설계에 필수는 아님
- 학습 기능을 구현하려면 추가적인 설계, 평가, 모니터링이 필요
- 학습 기법
  - 비모수적 학습
    - 파라미터를 바꾸지 않고도 자동으로 성능을 변경하고 개선하는 기법
  - 모수적 학습
    - 파라미터를 명시적으로 학습하거나 파인튜닝하는 기법
    - 감독 파인튜닝, 직접 선호 최적화
# 7.1 비모수적 기법
- 여러 가지 방법이 존재하며, 일반적인 몇 가지를 살펴봄
##### 7.1.1 비모수적 예시 학습
- 품질에 대한 측정 값을 제공하고 그 예시를 사용해 향후 성능 개선
  - 컨텍스트 내 학습을 위한 퓨샷
  - 가장 단순한 형태는 고정 퓨샷  -> 프롬프트에 하드코딩해 두고 변경하지 않음
- 모든 예제가 유용한 것은 아니면서 결국 비용과 지연 시간이 증가 -> 관련성이 높은 예제만 동적으로 제공
  - 상호작용의 컨텍스트, 수행한 액션, 받은 피드백 등을 저장하는 메모리 뱅크를 구축 -> 향후 행동을 이끄는 인간의 기억처럼 작동
  - 에이전트가 새로운 문제를 해결하기 위해 과거 사례 데이터베이스에서 정보를 추출
    - 문제 설명, 적용한 솔루션, 그 솔루션의 결과 포함
  - 성공적인 예제를 영속 저장소에 보관했다가 활용하면 성능이 크게 향상됨
    - 성공적 예제가 늘어나면 유형별, 텍스트 기반 검색, 의미론적 검색 등으로 추출
##### 7.1.2 리플렉시온
- 자기 비판 습관을 부여
  - 시도가 실패한 뒤에 무엇이 잘못되었고, 다음 시도를 어떻게 개선할지에 대해 짧은 성찰을 작성하는 방식
  - 성찰 내용은 과거 행동 및 성찰과 함께 메모리 버퍼에 저장
    - 새로운 시도를 시작하기 전에 가장 최근 성찰들을 다시 읽어 보면서 전략을 조정
